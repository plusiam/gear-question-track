# 톱니질문 트랙 (Gear-Question Track)

> 질문 카드를 **톱니처럼 끼워 연결**하며 *사실 → 추론 → 상상* 으로 이어지는 한 묶음 질문을 만드는 웹 도구.
> 특정 작품에 매이지 않고 **어떤 책·텍스트·장면에도** 쓸 수 있습니다.

## 🎯 목적
교사의 '톱니질문법'(생각을 한 칸씩 전환하는 발문)을 **학생이 스스로 질문 만드는 기술**로 전이시킵니다. 질문을 하나씩 던지고 끝내는 게 아니라, 한 장면에 **이어서 묻는** 경험을 줍니다.

## 👩‍🏫 학년·교과
- **대상**: 초등 5학년(모둠 활동 권장), 교사(전자칠판)
- **교과**: 도덕 · 국어(문해) — 2022 개정 교육과정
- **이론 바탕**: 라파엘(Raphael)의 **QAR**(질문-답 관계) — "답이 어디서 오는가"
  - 사실 = 책에 그대로(Right There) · 추론 = 책 바탕 추론(Think and Search·Author and You) · 상상 = 책 넘어 내 머릿속(On My Own)

## 🧩 사용 시나리오 (학생 주도 3단계)
1. **① 만들기** — 책 제목·장면을 적고, 떠오르는 질문을 자유롭게 만든다(발산).
2. **② 나누기** — "이 질문의 답이 어디 있지?"로 사실/추론/상상 분류.
3. **③ 잇기** — 분류한 질문 중 사실 → 추론 → 상상을 하나씩 골라 톱니로 잇고, 사이에 **'다리 한 줄'**(앞 답이 다음 질문의 디딤돌)을 적는다.

> 완성한 한 실은 **이미지(PNG)·인쇄**로 저장하거나, **결과 전체를 `.json`으로 저장·불러오기**(제출·이어하기·기기 이동)할 수 있습니다.

## ✨ 기능
- 책 제목·장면 직접 입력(범용) — 어떤 텍스트에도 재사용
- 드래그 / 탭 / 키보드 모두 지원, 색 + 라벨 이중 표기(색맹 대응)
- 정답·채점 없음 — 자기 점검·교사 확인 중심
- 작업 자동 저장(localStorage), 이어하기, JSON 내보내기/불러오기
- 모바일·태블릿·전자칠판 반응형

## 🔒 개인정보
학생 **개인정보를 수집하거나 외부로 보내지 않습니다.** 모든 작업은 기기 안(localStorage)에서만 처리됩니다.

## 💻 로컬에서 실행
```bash
# 이 폴더에서
python3 -m http.server
# 또는
npx serve
```
브라우저에서 `http://localhost:8000` 접속. (그냥 `index.html`을 더블클릭해도 대부분 동작하지만, 일부 브라우저는 로컬 서버를 권장합니다.)
